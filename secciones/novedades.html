A closure is a pairing of: A function and A reference to that function's outer scope (lexical environment) A lexical environment is part of every execution context (stack frame) and is a map between identifiers (i.e. local variable names) and values. Every function in JavaScript maintains a reference to its outer lexical environment. This reference is used to configure the execution context created when a function is invoked. This reference enables code inside the function to "see" variables declared outside the function, regardless of when and where the function is called. If a function was called by a function, which in turn was called by another function, then a chain of references to outer lexical environments is created. This chain is called the scope chain. In the following code, inner forms a closure with the lexical environment of the execution context created when foo is invoked, closing over variable secret: function foo() { const secret = Math.trunc(Math.random() * 100) return function inner() { console.log(`The secret number is ${secret}.`) } } const f = foo() // `secret` is not directly accessible from outside `foo` f() // The only way to retrieve `secret`, is to invoke `f` Expand snippet In other words: in JavaScript, functions carry a reference to a private "box of state", to which only they (and any other functions declared within the same lexical environment) have access. This box of the state is invisible to the caller of the function, delivering an excellent mechanism for data-hiding and encapsulation. And remember: functions in JavaScript can be passed around like variables (first-class functions), meaning these pairings of functionality and state can be passed around your program: similar to how you might pass an instance of a class around in C++. If JavaScript did not have closures, then more states would have to be passed between functions explicitly, making parameter lists longer and code noisier. So, if you want a function to always have access to a private piece of state, you can use a closure. ...and frequently we do want to associate the state with a function. For example, in Java or C++, when you add a private instance variable and a method to a class, you are associating the state with functionality. In C and most other common languages, after a function returns, all the local variables are no longer accessible because the stack-frame is destroyed. In JavaScript, if you declare a function within another function, then the local variables of the outer function can remain accessible after returning from it. In this way, in the code above, secret remains available to the function object inner, after it has been returned from foo.

como sacar provecho al algoritmo

¿Qué es el algoritmo?
Un algoritmo es un código basado en reglas matemáticas que sirve para llevar a cabo una actividad siguiendo unas reglas bien definidas o unas instrucciones.
En este caso el algoritmo se encarga de establecer el orden en que vas a ver las publicaciones de los usuarios que sigues cada vez que entres en la aplicación de esta red social.

¿Cómo funciona el algoritmo?
Aunque hablemos de algoritmo, en singular, Instagram ha confirmado que existen varios algoritmos que se encargan de escoger esos contenidos que vemos en cada canal de Instagram.
El algoritmo se basa en el engagement, es decir, en las interacciones del usuario con las diferentes cuentas que sigues, por lo que deberás de ser tú, a través de tus interacciones, el que le digas al algoritmo qué te interesa para que te muestre lo que quieres ver. Vamos a ver todo lo que necesitas saber para entender mejor este algoritmo y descubrir que realmente no tiene nada en contra de nadie, sino que solo trata de mostrar a los usuarios lo que considera más interesantes para ellos
(al final, Instagram lo que quiere es que pasemos tiempo en su red social y si tu contenido ayuda a ello, lo potenciará).

¿Qué pasa cuando publicas un contenido en Instagram?
Cuando publicas un contenido nuevo, Instagram se lo muestra a un porcentaje de tus seguidores para que, con sus interacciones, pueda saber si se trata de un contenido de valor o no.

Al final, lo que quiere el algoritmo es que pasemos tiempo de calidad en la aplicación.
Si los usuarios que ven ese contenido interactúan con él, el algoritmo entiende que es interesante y se lo irá mostrando a más usuarios (tanto de los que te siguen como los que puedan llegar a través de los hashtags o la ubicación que has añadido a tu publicación). Mientras siga habiendo interacciones, seguirá aumentando el alcance de tu publicación.
Si los usuarios no interactúan, el algoritmo entiende que el contenido no es interesante y deja poco a poco de mostrarlo, por lo que los usuarios a los que el algoritmo no les ha mostrado la publicación, tendrán que acceder a tu perfil para verlo (aunque tú también puedes realizar algunas acciones para intentar darle más alcance y que el algoritmo lo muestre).

Si has sufrido una penalización por parte de Instagram, puedes ver reducido el alcance de tus publicaciones drásticamente.
Debido a esto, tus publicaciones no alcanzan a todos tus usuarios, sino que alcanzan a una parte de éstos, normalmente a tus seguidores más fieles. De ahí la importancia de crear comunidad en Instagram y no simplemente conseguir seguidores.
